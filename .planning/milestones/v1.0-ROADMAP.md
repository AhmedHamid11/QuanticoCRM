# Milestone v1.0: Platform Update System

**Status:** SHIPPED 2026-02-01
**Phases:** 1-5
**Total Plans:** 9

## Overview

Enable versioned platform updates that orgs can adopt on their own schedule with full transparency. Updates are additive - existing org data is never altered. All orgs stay on the same version (auto-updated on deploy).

## Phases

### Phase 1: Platform Versioning
**Goal**: Track platform schema version and each org's current version.
**Plans**: 2 plans

- [x] 01-01-PLAN.md — Database schema and version comparison service
- [x] 01-02-PLAN.md — Version API endpoints

**Delivered:** Version tracking infrastructure with semver comparison, platform_version table, org current_version column, and REST API endpoints.

---

### Phase 2: Change Tracking
**Goal**: Record what changed between platform versions.
**Plans**: 1 plan

- [x] 02-01-PLAN.md — Changelog package and API endpoints

**Delivered:** Structured changelog system with category-based entries (Added, Changed, Fixed, etc.) and API to query changes between versions.

---

### Phase 3: Changelog UI
**Goal**: Let org admins see what changed in each version.
**Plans**: 1 plan

- [x] 03-01-PLAN.md — Changelog page and admin link

**Delivered:** Admin panel changelog page showing version history with categorized changes.

---

### Phase 4: Update Propagation
**Goal**: Automatically update all org databases after deploy.
**Plans**: 3 plans

- [x] 04-01-PLAN.md — Migration tracking schema, entity, and repository
- [x] 04-02-PLAN.md — MigrationPropagator service and startup integration
- [x] 04-03-PLAN.md — API endpoints and changelog page migration status UI

**Delivered:** Blocking migration on startup, sequential org processing with skip-and-continue on failures, admin visibility into migration status, manual retry for failed orgs.

---

### Phase 5: New Org Provisioning
**Goal**: New orgs automatically get the latest platform version.
**Plans**: 2 plans

- [x] 05-01-PLAN.md — Wire VersionRepo into AuthService for version-aware org creation
- [x] 05-02-PLAN.md — Fix Register() to include version lookup (gap closure)

**Delivered:** Version-aware org provisioning - new orgs stamped with current platform version in both CreateOrganization and Register flows.

---

## Milestone Summary

**Key Decisions:**
- Blocking migration on startup (vs background) for data consistency
- Skip-and-continue for failed orgs to not block healthy orgs
- Platform admin visibility for migration status with manual retry

**Issues Resolved:**
- Idempotent migrations for duplicate column errors
- SQL comment stripping for Railway deployment
- Embedded migrations in binary for deployment

**Technical Debt:**
- None identified
