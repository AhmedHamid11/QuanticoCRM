## E2E Flow Verification

### Flow 1: Login (PASS)

```
1. Request → CORS validates origin
2. Request → Rate limiter checks (5/min)
3. Handler → Validates credentials
4. Handler → Sets HttpOnly refresh cookie
5. Handler → Returns access token (memory only)
6. Handler → Audit logs LOGIN_SUCCESS
```

### Flow 2: Session Management (PASS)

```
1. Login → Session created with timestamps
2. Request → Session middleware updates last_activity_at (fire-and-forget)
3. Frontend → Tracks remaining time via session.svelte.ts
4. Frontend → Shows 5-minute warning toast
5. User → Clicks "Stay logged in" → POST /auth/extend-session
6. Timeout → 401 SESSION_EXPIRED → Frontend redirects to login
```

### Flow 3: Admin Action Audit (PASS)

```
1. Admin → Changes user role
2. Handler → Captures old role before update
3. Handler → Updates role in database
4. Handler → Audit logs ROLE_CHANGE with old/new values
5. Admin → Views audit log UI → Sees role change entry
6. Admin → Exports to CSV/JSON → Complete audit trail
```

### Flow 4: Forced Password Change (PASS)

```
1. User → Logs in with weak password
2. Service → Detects weak password (isPasswordWeak)
3. Service → Sets mustChangePassword=true in JWT
4. User → Attempts protected endpoint
5. Middleware → Returns 403 PASSWORD_CHANGE_REQUIRED
6. Frontend → Intercepts, redirects to /change-password
7. User → Changes password
8. Handler → Audit logs PASSWORD_CHANGE
9. User → Re-authenticated with mustChangePassword=false
```
