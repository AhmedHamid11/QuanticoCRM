## Tech Debt

### Phase 12: Real-Time Detection

- **IsBlockMode hardcoded to false** (realtime.go:87) — matching_rules table doesn't have block_mode column. Block mode works at the UI level (frontend can enforce "type DUPLICATE" confirmation) but the backend doesn't enforce preventing saves. This is documented in code.

### Phase 16: Admin UI

- **Inline type definitions** in duplicate-rules and scan-jobs pages instead of importing from data-quality.ts. Intentional design decision (plans executed in parallel, couldn't guarantee data-quality.ts existed yet). Non-impactful — types are consistent.

### Phase 11: Detection Foundation

- **DETECT-07 partial** — Negative signals framework (e.g., different company: -30% score, different email domain: -20% score) has the weighted scoring foundation ready but specific negative signal logic not implemented. Can be added by extending scorer.go without architectural changes.
