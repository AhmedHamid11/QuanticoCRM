# Requirements Archive: Quantico CRM v4.0 Salesforce Merge Integration

**Archived:** 2026-02-10
**Status:** All v4.0 requirements shipped

## v4.0 Requirements — All Complete

### Merge Instruction Payload

- [x] **SFI-01**: Quantico builds merge instruction JSON from dedup resolution (winner_id, loser_id, field_values) — Phase 17
- [x] **SFI-02**: Merge instruction includes all mapped fields for the object — Phase 17
- [x] **SFI-03**: Merge instructions use 18-character Salesforce Record IDs (not 15-char) — Phase 17
- [x] **SFI-04**: Merge instructions use Salesforce field API names, not display labels — Phase 17

### Salesforce Authentication

- [x] **SFI-05**: Admin can configure Salesforce Connected App (Client ID, Client Secret, Redirect URL) — Phase 17
- [x] **SFI-06**: Quantico initiates OAuth 2.0 authorization flow (user grants scope) — Phase 17
- [x] **SFI-07**: Quantico stores encrypted OAuth tokens in database (refresh token, access token, expiry) — Phase 17
- [x] **SFI-08**: Quantico refreshes access tokens before expiry (proactive, not reactive) — Phase 17
- [x] **SFI-09**: Auth failures surface clear error messages (invalid credentials, scope issues, etc.) — Phase 17

### Batch Delivery

- [x] **SFI-10**: Quantico accepts merge instructions from dedup system (manual trigger or scheduled job) — Phase 17
- [x] **SFI-11**: Quantico groups merge instructions into batches (up to 200 per batch) — Phase 17
- [x] **SFI-12**: Quantico generates unique batch_id (format: QTC-YYYYMMDD-NNN) and instruction_ids (MI-NNNN) — Phase 17
- [x] **SFI-13**: Quantico POSTs merge instructions to Salesforce staging object via REST API — Phase 17
- [x] **SFI-14**: Quantico handles Salesforce API errors gracefully — Phase 17

### Rate Limiting

- [x] **SFI-15**: Quantico tracks API usage per org (calls per 24-hour period) — Phase 18
- [x] **SFI-16**: Quantico respects Salesforce API limits (100K calls/day for Enterprise) — Phase 18
- [x] **SFI-17**: Quantico implements exponential backoff (5s, 10s, 20s, 40s) on 429 Too Many Requests — Phase 18
- [x] **SFI-18**: Quantico pauses batch delivery at 80% capacity — Phase 18
- [x] **SFI-19**: Quantico allows manual trigger to override rate limiting — Phase 18

### Audit Logging

- [x] **SFI-20**: Quantico logs every merge instruction sent (batch_id, instruction_id, timestamp, winner_id, loser_id) — Phase 19
- [x] **SFI-21**: Quantico logs delivery status (success, error, retry) with Salesforce response details — Phase 19
- [x] **SFI-22**: Quantico stores audit logs with 7-year retention (SOX compliance) — Phase 19
- [x] **SFI-23**: Admin can query audit logs by batch_id, date range, org, result status — Phase 19

### Admin Configuration UI

- [x] **SFI-24**: Admin page allows connecting Salesforce org (OAuth flow, test connection) — Phase 17 + 19
- [x] **SFI-25**: Admin page shows connection status (connected, disconnected, token expired) — Phase 17
- [x] **SFI-26**: Admin page allows enabling/disabling sync for an org — Phase 17
- [x] **SFI-27**: Admin page displays manual trigger for immediate merge delivery — Phase 17
