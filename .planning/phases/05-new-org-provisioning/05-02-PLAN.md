The Register() function (line 92) calls `s.repo.CreateOrganization` directly in TWO places:
- Line 129: When `s.tenantProvisioning != nil` (tenant database path)
- Line 150: Legacy path without tenant provisioning

Both paths bypass the version lookup logic that exists in `CreateOrganization` service method (lines 838-849).

**Fix: Add version lookup ONCE, before the if/else branch, and use it in both paths.**

Insert the following code BEFORE line 125 (`// Create organization with tenant database`):

```go
// Get current platform version for new org
platformVersion := "v0.1.0" // Default fallback
if s.versionRepo != nil {
    pv, err := s.versionRepo.GetPlatformVersion(ctx)
    if err != nil {
        log.Printf("[Register] Warning: failed to get platform version: %v, using default", err)
    } else {
        platformVersion = pv.Version
        log.Printf("[Register] New org will be created at platform version %s", platformVersion)
    }
}
```

Then update BOTH `s.repo.CreateOrganization` calls to include the version:

**Line 129-132 (tenant path):**
```go
org, err = s.repo.CreateOrganization(ctx, entity.OrganizationCreateInput{
    Name:           input.OrgName,
    Slug:           slug,
    CurrentVersion: platformVersion,
})
```

**Line 150-153 (legacy path):**
```go
org, err = s.repo.CreateOrganization(ctx, entity.OrganizationCreateInput{
    Name:           input.OrgName,
    Slug:           slug,
    CurrentVersion: platformVersion,
})
```

This mirrors the pattern used in CreateOrganization (lines 838-849) but applies it to the Register flow.
