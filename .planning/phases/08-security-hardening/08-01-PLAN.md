  <name>Task 2: Create body limit middleware</name>
  <files>fastcrm/backend/internal/middleware/bodylimit.go</files>
  <action>
Create bodylimit.go with configurable request body size limits.

Environment variables (with defaults):
- MAX_BODY_SIZE: default 1MB (1048576 bytes)
- MAX_UPLOAD_SIZE: default 10MB (10485760 bytes)

Export:
- `DefaultBodyLimit int` - 1MB
- `UploadBodyLimit int` - 10MB (read from env on init)
- `BodyLimit(limit int) fiber.Handler` - returns middleware that checks Content-Length and returns 413 if exceeded

The 413 response format:
```json
{
  "error": "Request body too large",
  "limit": 1048576,
  "unit": "bytes"
}
```

Use `c.Request().Header.ContentLength()` to check size before parsing.

Note: Fiber's built-in limiter reads the entire body. We want to reject BEFORE reading to prevent memory exhaustion. Check Content-Length header only (attackers can lie, but this catches honest large requests; malicious requests are handled by read timeout).

The import handler internally uses POST for file uploads. We need to add the upload limit to the import routes specifically. Check if RegisterRoutes accepts a group - if so, create a subgroup with upload limit.

Actually, the cleaner approach: Apply default limit globally, then specific routes can override. But Fiber doesn't support overriding. Instead:
- Apply default 1MB limit at app level (after recovery/logger, before routes)
- Use Fiber's built-in limiter for the app-wide default
- The body limit middleware we created is for route-specific overrides if needed

Simplified approach:
```go
// SECURITY: Default body limit 1MB (HARD-06)
app.Use(limiter.New(limiter.Config{
    Max: 0, // Not using request count limit here
    ...
}))
```

Wait, that's rate limiting. For body limit, use Fiber's BodyLimit config in fiber.New():

```go
app := fiber.New(fiber.Config{
    BodyLimit: middleware.DefaultBodyLimit, // 1MB default
    ErrorHandler: ...
})
```

Then for upload routes, use our BodyLimit middleware to allow larger:
```go

importGroup := protected.Group("/import", middleware.BodyLimit(middleware.UploadBodyLimit))
```

Actually, looking at the code, importHandler.RegisterRoutes takes a router group. We can create a subgroup with different limit.

Simplest approach that works:
1. Set app-wide BodyLimit in fiber.Config to 10MB (max we'll allow)
2. Apply our BodyLimit(1MB) middleware to all routes EXCEPT import
3. Import routes get the default 10MB from app config
