  <files>FastCRM/fastcrm/backend/internal/changelog/entries.go</files>
  <action>
Create the changelog package with:

1. **Category type and constants** following Keep a Changelog convention:
   - CategoryAdded, CategoryChanged, CategoryFixed, CategoryRemoved, CategoryDeprecated, CategorySecurity

2. **Entry struct** with Category and Description fields (JSON tags for API serialization)

3. **VersionChangelog struct** with Version string and Entries slice

4. **Entries map** (var Entries = map[string][]Entry) with initial changelog for v0.1.0:
   - Added: "Initial platform version with core CRM entities"
   - Added: "Contact, Account, Task, Quote entity support"
   - Added: "Multi-tenant database architecture"

5. **GetSortedVersions()** function:
   - Extracts all version keys from Entries map
   - Sorts descending (newest first) using semver.Compare
   - Returns []string

6. **GetEntriesForVersion(version string)** function:
   - Looks up version in Entries map
   - Returns ([]Entry, bool) - entries and whether found

7. **GetEntriesBetweenVersions(fromVersion, toVersion string)** function:
   - Iterates GetSortedVersions()
   - Includes versions where: version > fromVersion AND version <= toVersion
   - Returns []VersionChangelog ordered newest to oldest

Import golang.org/x/mod/semver (already in go.mod from Phase 1).

IMPORTANT: All version keys in Entries map must have v prefix (e.g., "v0.1.0" not "0.1.0") to match Phase 1 version format decisions.

<task type="auto">
  <name>Task 2: Extend version handler with changelog endpoints</name>
  <files>FastCRM/fastcrm/backend/internal/handler/version.go</files>
  <action>
Extend the existing VersionHandler with two new endpoints:

1. **GetChangelog endpoint** (GET /api/v1/version/changelog):
   - Query param: ?version=v0.2.0 (optional, defaults to current platform version)
   - If version not provided, get platform version from h.repo.GetPlatformVersion()
   - Normalize version using h.service.Normalize()
   - Call changelog.GetEntriesForVersion(version)
   - Return JSON: {"version": "v0.1.0", "entries": [...]}
   - If version has no entries, return {"version": "...", "entries": []} (empty array, not error)

2. **GetChangelogSince endpoint** (GET /api/v1/version/changelog/since):
   - Query param: ?from=v0.1.0 (required)
   - If missing, return 400 with {"error": "Missing 'from' query parameter"}
   - Normalize fromVersion using h.service.Normalize()
   - Get current platform version as toVersion via h.repo.GetPlatformVersion()
   - Call changelog.GetEntriesBetweenVersions(fromVersion, toVersion)
   - Return JSON: {"fromVersion": "v0.1.0", "toVersion": "v0.2.0", "changelogs": [...]}

3. **Register routes** in RegisterRoutes():
   - Add: version.Get("/changelog", h.GetChangelog)
   - Add: version.Get("/changelog/since", h.GetChangelogSince)

Add import for the new changelog package:
"github.com/fastcrm/backend/internal/changelog"

Follow existing handler patterns (error handling, JSON responses) from GetPlatformVersion and GetCurrentVersion.
