```go
package entity

// ImportDuplicateMatch represents an import row that matched an existing database record
type ImportDuplicateMatch struct {
    ImportRowIndex  int                    `json:"importRowIndex"`  // 0-based row index in CSV
    ImportRow       map[string]interface{} `json:"importRow"`       // The import row data
    MatchedRecordID string                 `json:"matchedRecordId"` // ID of the matched existing record
    MatchedRecord   map[string]interface{} `json:"matchedRecord"`   // The matched record data
    ConfidenceScore float64                `json:"confidenceScore"` // 0.0-1.0 overall match score
    ConfidenceTier  string                 `json:"confidenceTier"`  // "high", "medium", "low"
    MatchedFields   []string               `json:"matchedFields"`   // Which fields matched (for UI highlighting)
    RuleName        string                 `json:"ruleName"`        // Which matching rule produced this match
    OtherMatches    []ImportMatchCandidate `json:"otherMatches,omitempty"` // Additional lower-score matches
}

// ImportMatchCandidate represents an alternative match for a row (not the top match)
type ImportMatchCandidate struct {
    RecordID string  `json:"recordId"`
    Name     string  `json:"name"` // Display name for the record
    Score    float64 `json:"score"`
}

// ImportDuplicateGroup represents rows within the CSV that duplicate each other
type ImportDuplicateGroup struct {
    GroupID    string                   `json:"groupId"`    // Unique group identifier (hash)
    RowIndices []int                    `json:"rowIndices"` // 0-based row indices of duplicate rows
    Rows       []map[string]interface{} `json:"rows"`       // The duplicate row data
    KeepIndex  int                      `json:"keepIndex"`  // Default: first row index (auto-selected)
}

// DuplicateCheckResult is the combined result of database + within-file detection
type DuplicateCheckResult struct {
    DatabaseMatches  []ImportDuplicateMatch `json:"databaseMatches"`
    WithinFileGroups []ImportDuplicateGroup `json:"withinFileGroups"`
    TotalRows        int                    `json:"totalRows"`        // Total rows in the CSV
    FlaggedRows      int                    `json:"flaggedRows"`      // Rows that need review
}

// ImportResolution represents the user's decision for a flagged row
type ImportResolution struct {
    Action          string `json:"action"`                    // "skip", "update", "import", "merge"
    SelectedMatchID string `json:"selectedMatchId,omitempty"` // Which match to act on (for update/merge)
}
```

Key design notes:
- ConfidenceTier included so frontend doesn't need to recompute tier from score
- KeepIndex on groups defaults to first row index per research recommendation
- ImportResolution will be used by Plan 14-03 when processing the import with resolutions
- OtherMatches on ImportDuplicateMatch allows user to switch which existing record they compare against

Constructor:

```go
type ImportDuplicateService struct {
    detector         *dedup.Detector
    matchingRuleRepo *repo.MatchingRuleRepo
}

func NewImportDuplicateService(detector *dedup.Detector, matchingRuleRepo *repo.MatchingRuleRepo) *ImportDuplicateService {
    return &ImportDuplicateService{
        detector:         detector,
        matchingRuleRepo: matchingRuleRepo,
    }
}
```

Main method - `CheckDuplicates(ctx, dbConn db.DBConn, orgID, entityType string, importRows []map[string]interface{}) (*entity.DuplicateCheckResult, error)`:
1. Call `detectDatabaseDuplicates()` to find matches against existing DB records
2. Call `detectWithinFileDuplicates()` to find rows that duplicate each other within the CSV
3. Calculate flaggedRows count = unique row indices from database matches + rows in within-file groups
4. Return DuplicateCheckResult

**detectDatabaseDuplicates** method:
- Get tenant-routed matchingRuleRepo via `matchingRuleRepo.WithDB(dbConn)`
- For each import row, call `detector.CheckForDuplicates(ctx, dbConn, orgID, entityType, importRow, "")` (excludeID = "" since import rows have no ID yet)
- For each match returned, fetch the matched record via `util.FetchRecordAsMap()`
- Build ImportDuplicateMatch with:
  - Top match as primary (highest score)
  - Other matches as OtherMatches array
  - MatchedFields from matchResult.MatchingFields
  - ConfidenceTier from matchResult.ConfidenceTier
  - Record name extracted from matched record (try "name", then "firstName"+" "+"lastName")
- Deduplicate: if same import row matches via multiple runs, keep the highest-score match
- IMPORTANT: The Detector already handles blocking + scoring internally. Don't re-implement that logic. Just call Detector.CheckForDuplicates() per import row.

**detectWithinFileDuplicates** method:
- Get enabled rules via matchingRuleRepo.WithDB(dbConn).ListEnabledRules(ctx, orgID, entityType)
- For each row, create a hash key from normalized values of all match fields across all rules
- Hash method: for each rule's field configs, extract field value from row, normalize it (lowercase, trim; for phone fields strip non-digits), concatenate sorted normalized values with "|" separator, SHA-256 hash the result
- Group rows by hash: hash -> []rowIndex
- Any hash with 2+ indices = duplicate group
- Create ImportDuplicateGroup for each group, set KeepIndex to first row index as default
