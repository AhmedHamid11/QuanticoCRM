Create the duplicate rules management page using Svelte 5 runes and Tailwind CSS. This page consumes the existing backend API at GET/POST/PUT/DELETE `/dedup/rules`.

NOTE: If Plan 01 has not completed yet (data-quality.ts not available), import directly from `$lib/utils/api` using `get`, `post`, `put`, `del` and define types inline. If data-quality.ts exists, import from `$lib/api/data-quality`.

**Page structure (per locked decisions):**

1. **Header bar:** Title "Duplicate Rules", entity type filter dropdown (fetch entities from `/admin/entities`), "New Rule" button.

2. **Rules table:** Table with columns: Name, Entity Type, Status (enabled/disabled toggle), Priority, Threshold, Fields, Actions (delete).
   - Click on any row to expand inline editing form below that row (per user decision: "table of rules per entity type, click to expand inline for editing")
   - Only one row expanded at a time (collapse previous when new one opens)

3. **Inline edit form (expanded row):**
   - Name: text input
   - Description: text input
   - Entity Type: dropdown (read-only on existing rules)
   - Is Enabled: toggle
   - Priority: number input
   - Blocking Strategy: dropdown (options: soundex, prefix, domain, phone, none)
   - **Field configurations section** (per user decision: "Field configuration via dropdown: select field, choose matching type, set weight"):
     - List of field config rows, each with:
       - Field name: dropdown populated from entity's fields (GET `/admin/entities/{entity}/fields`)
       - Match type: dropdown (exact, fuzzy, phonetic)
       - Weight: number input (0-1, step 0.1)
       - Remove button (X)
     - "Add Field" button to add new field config row
   - **Confidence thresholds section** (per user decision: "numeric inputs with color-coded tier labels"):
     - Overall threshold: number input (0-1)
     - High confidence threshold: number input with red badge label "High"
     - Medium confidence threshold: number input with yellow badge label "Medium"
     - Low (implicit, below medium) shown with blue badge label "Low"
     - Use `getConfidenceBadgeClass` from existing dedup.ts for tier colors
   - Save button (Enter key to save per research recommendation, plus explicit Save button for clarity)
   - Cancel button (collapses the row)

4. **Test Rule button** (per user decision: "runs the rule against existing data and shows sample matches with scores"):
   - Visible in expanded edit form
   - When clicked, calls POST `/dedup/{entityType}/check` with sample data
   - Actually: run a lightweight test by sending a dummy record to the check endpoint, OR better: call the existing detection endpoint with a representative record
   - Show results in a small results panel below the form: list of matched records with scores
   - Show "Testing..." loading state while running

5. **New Rule form:**
   - Reuse the same inline edit form, but at the top of the table as a new expandable row
   - On save, POST to `/dedup/rules` with MatchingRuleCreateInput

6. **Delete confirmation:**
   - Click delete icon on a rule row
   - Show `confirm()` dialog: "Delete rule '{name}'? This cannot be undone."
   - On confirm, DELETE `/dedup/rules/{id}`
   - Optimistic UI: remove from list immediately, restore on error
