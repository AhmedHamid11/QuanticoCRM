Create the review queue page using Svelte 5 runes and Tailwind CSS. This page consumes the ListPendingAlerts endpoint added in Plan 01.

**Page structure (per locked decisions):**

1. **Header bar:**
   - Title "Review Queue"
   - Entity type filter dropdown (all entity types / Contact / Account / Lead / etc.)
   - Count badge showing total pending alerts
   - Pagination controls (if more than one page)

2. **Card list** (per user decision: "Card-based groups -- each duplicate group displayed as a card showing matched records with confidence score"):
   - Each pending alert becomes a card
   - Default sort: by confidence score highest first (already sorted by backend query)
   - Each card has:
     - **Left border color** based on confidence tier: red (high), yellow (medium), blue (low) -- use `getBannerClass` from existing dedup.ts
     - **Checkbox** on the left for bulk selection
     - **Header:** Entity type label + record identifier (first match's name or ID)
     - **Confidence badge:** e.g., "95% High" with tier-colored badge using `getConfidenceBadgeClass`
     - **Matched records list:** Show the source record and each matched record with:
       - Record name/identifier (from matches array recordName or recordId)
       - Individual match score
       - Key matching fields highlighted
     - **Card should show enough info to make quick dismiss/merge decisions** (per specifics)
     - **Action buttons** (right side, per user decision):
       - "Dismiss" button (gray, marks as not duplicates via POST resolve-alert with status "dismissed")
       - "Quick Merge" button (blue, auto-pick best fields per research: uses merge preview's suggestedSurvivorID and auto-selects most complete field values, then executes merge)
       - "Merge" button (outlined, navigates to full merge wizard at `/admin/data-quality/merge/{alertId}`)

3. **Quick Merge logic:**
   - Call POST `/merge/preview` with recordIDs (source record + all match recordIDs) and entityType
   - Use `suggestedSurvivorID` from preview as survivor
   - Auto-select merged field values: for each field, pick value from survivor (most complete record per completeness scores)
   - Call POST `/merge/execute` with auto-built MergeRequest
   - On success: remove card from list with optimistic UI, show toast "Records merged. Undo?" with clickable Undo link
   - On error: show error toast

4. **Floating bulk action bar** (per user decision: "Checkbox selection on each card with floating bulk action bar when items selected"):
   - Fixed to bottom of screen when `selectedIds.size > 0`
   - Shows: "{N} selected" count
   - Two buttons: "Merge All" and "Dismiss All"
   - Dismiss All: resolves each selected alert as dismissed (sequential API calls, progress indicator)
   - Merge All: for each selected alert, perform Quick Merge logic (sequential, with progress count like "Merging 3/7...")
   - On completion: toast with result count, clear selection

5. **Empty state:**
   - When no pending alerts: show centered illustration with "No duplicates found" message
   - Subtitle: "All clear! No duplicate records detected in your data."
   - Optional call to action: "Run a scan" button linking to scan jobs page

6. **Loading state:**
   - Show skeleton cards while loading (3 placeholder cards with pulse animation)

**State management (Svelte 5 runes):**
```typescript

let bulkProgress = $state({ current: 0, total: 0 });
```

**Pagination:** Load next page on click, update URL params.
