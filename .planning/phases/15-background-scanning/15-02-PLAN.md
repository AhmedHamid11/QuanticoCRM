**ScanJobService struct:**
```go
type ScanJobService struct {
    detector         *dedup.Detector
    scanJobRepo      *repo.ScanJobRepo
    pendingAlertRepo *repo.PendingAlertRepo
    matchingRuleRepo *repo.MatchingRuleRepo
    dbManager        *db.Manager
    authRepo         *repo.AuthRepo        // For getting org users (notifications)

    // Per-tenant rate limiting (max 2 concurrent jobs per org)
    runningJobs map[string]int // orgID -> count of running jobs
    runningMu   sync.Mutex

    // Progress event callback (set by handler for SSE broadcasting)
    onProgress func(event ProgressEvent)
    progressMu sync.RWMutex
}
```

**ProgressEvent type** (exported, used by handler for SSE):
```go
type ProgressEvent struct {
    JobID            string `json:"jobId"`
    OrgID            string `json:"orgId"`
    EntityType       string `json:"entityType"`
    ProcessedRecords int    `json:"processedRecords"`
    TotalRecords     int    `json:"totalRecords"`
    DuplicatesFound  int    `json:"duplicatesFound"`
    Status           string `json:"status"`
}
```

**NewScanJobService constructor** takes detector, scanJobRepo, pendingAlertRepo, matchingRuleRepo, dbManager, authRepo. Initializes runningJobs map.

**SetProgressCallback(fn func(ProgressEvent))** — called by handler to wire SSE broadcasting.

**ExecuteScan(ctx context.Context, orgID, entityType, triggerType string, scheduleID *string) (string, error):**

1. **Rate limit check:** Lock runningMu, check `runningJobs[orgID] >= 2`. If at limit, return error "max concurrent jobs reached for org". Otherwise increment count, defer decrement.

2. **Resolve tenant DB:** Use dbManager.GetDB(orgID) to get the tenant's database connection. If error, return.

3. **Count total records:** Query `SELECT COUNT(*) FROM {tableName} WHERE org_id = ?` to get total.

4. **Create job record:** Create ScanJob entity with status "running", TotalRecords set, StartedAt = now. Use scanJobRepo.CreateJob(ctx, job). Use sfid.NewScanJob() for ID.

5. **Check for existing checkpoint** (resume from failure): scanJobRepo.GetCheckpoint(ctx, jobID). If exists, set offset = checkpoint.LastOffset.

6. **Chunk loop** (chunkSize = 500):
   ```
   for {
       // Check context cancellation (graceful shutdown)
       select {
       case <-ctx.Done():
           scanJobRepo.UpdateJobStatus(ctx, jobID, "cancelled")
           return jobID, ctx.Err()
       default:
       }

       // Fetch chunk using ORDER BY id LIMIT ? OFFSET ?
       records := fetchChunk(tenantDB, tableName, orgID, chunkSize, offset)
       if len(records) == 0 { break }

       // Process chunk: for each record, call detector.CheckForDuplicates
       // Store matches as PendingDuplicateAlerts (reuse Phase 12 pattern)
       duplicatesInChunk := processChunk(ctx, tenantDB, orgID, entityType, records)

       // Update progress in DB
       offset += len(records)
       totalDuplicates += duplicatesInChunk
       scanJobRepo.UpdateJobProgress(ctx, jobID, offset, totalDuplicates)

       // Save checkpoint after EVERY chunk
       saveCheckpoint(ctx, jobID, offset)

       // Emit progress event for SSE listeners
       emitProgress(ProgressEvent{
           JobID: jobID, OrgID: orgID, EntityType: entityType,
           ProcessedRecords: offset, TotalRecords: totalRecords,
           DuplicatesFound: totalDuplicates, Status: "running",
       })

       // Sleep 100ms between chunks to allow WAL checkpoint window
       // (per RESEARCH.md Pitfall #4: WAL checkpoint starvation)
       time.Sleep(100 * time.Millisecond)

       if len(records) < chunkSize { break }
   }
   ```

7. **On chunk failure** (error during processChunk):
   - Get checkpoint, check RetryCount
   - If RetryCount < 1: increment retry count, re-attempt the same chunk
   - If RetryCount >= 1: save checkpoint, mark job "failed", set ErrorMessage, emit failure progress event
   - Per CONTEXT.md: "Auto-retry the failed chunk once; if it fails again, save checkpoint and mark job as failed"

8. **On completion:** Mark job "completed", set CompletedAt, emit final progress event with status "completed".

**processChunk helper:** For each record in chunk, call `detector.CheckForDuplicates(ctx, tenantDB, orgID, entityType, record, recordID)`. For each match found, create a PendingDuplicateAlert using pendingAlertRepo (reuse the alert upsert pattern from Phase 12). Use canonical pair keys (smaller ID first) to avoid storing A->B and B->A. Track duplicates count and return it.

**fetchChunk helper:** Execute `SELECT * FROM {tableName} WHERE org_id = ? ORDER BY id LIMIT ? OFFSET ?` and use `util.ScanRowsToMaps` to get []map[string]interface{}.

**emitProgress helper:** Lock progressMu read, call onProgress callback if set.

**ResumeInterruptedJobs(ctx context.Context):** Called at startup. Query all jobs with status "running" (from crash/restart). For each, mark as "failed" (they can be manually retried). Don't auto-resume to avoid thundering herd on restart.

**RetryJob(ctx context.Context, orgID, jobID string) (string, error):** Create a NEW job that inherits the entity type from the failed job. Copy the checkpoint offset so it resumes from where the failed job left off. Set trigger_type = "manual". Per CONTEXT.md: "Retry resumes from last checkpoint".

**CanRunJob(orgID string) bool:** Check per-tenant rate limit (exported for handler use).

IMPORTANT considerations:
- Use `context.Background()` with 30-second timeout per chunk (not Fiber context, which gets pooled). Per Phase 12 decision.
- Add panic recovery in the goroutine wrapper (per Phase 12 decision).
- Store detected duplicates as PendingDuplicateAlerts so they appear in the review queue (per CONTEXT.md: "Scan results feed into the central duplicate review queue").
- Partial results are stored as alerts during execution, so they're visible immediately even if the job fails midway (per CONTEXT.md: "Partial results from failed scans are visible immediately").

**ScanScheduler struct:**
```go
type ScanScheduler struct {
    scheduler   gocron.Scheduler
    scanService *ScanJobService
    scanJobRepo *repo.ScanJobRepo
    dbManager   *db.Manager

    // Track gocron job UUIDs for removal when schedules change
    activeJobs map[string]gocron.Job // key: "{orgID}:{entityType}" -> gocron.Job
    mu         sync.Mutex
}
```

**NewScanScheduler(scanService, scanJobRepo, dbManager):** Create gocron.Scheduler with `gocron.NewScheduler()`. Initialize activeJobs map.

**Start() error:**
1. Load all enabled schedules from DB: `scanJobRepo.ListAllEnabledSchedules(ctx)`
2. For each schedule, call `registerSchedule(schedule)`
3. Start the gocron scheduler: `s.scheduler.Start()`
4. Log total registered schedules

**Shutdown() error:** Call `s.scheduler.Shutdown()`. This gracefully stops all scheduled jobs.

**registerSchedule(schedule entity.ScanSchedule) error:**
Build gocron job definition based on frequency:

```go
key := schedule.OrgID + ":" + schedule.EntityType

var jobDef gocron.JobDefinition
switch schedule.Frequency {
case entity.ScanFrequencyDaily:
    jobDef = gocron.DailyJob(1, gocron.NewAtTimes(
        gocron.NewAtTime(uint(schedule.Hour), uint(schedule.Minute), 0),
    ))
case entity.ScanFrequencyWeekly:
    weekday := time.Weekday(*schedule.DayOfWeek)
    jobDef = gocron.WeeklyJob(1, gocron.NewWeekdays(weekday), gocron.NewAtTimes(
        gocron.NewAtTime(uint(schedule.Hour), uint(schedule.Minute), 0),
    ))
case entity.ScanFrequencyMonthly:
    jobDef = gocron.MonthlyJob(1, gocron.NewDaysOfTheMonth(*schedule.DayOfMonth), gocron.NewAtTimes(
        gocron.NewAtTime(uint(schedule.Hour), uint(schedule.Minute), 0),
    ))
}
```

Create the task function that:
1. Checks if a job is already running for this entity (per CONTEXT.md: "skip the new run and log that it was skipped")
2. If running, log the skip and return
3. Otherwise call `scanService.ExecuteScan(ctx, orgID, entityType, "scheduled", &scheduleID)` in a goroutine with panic recovery
4. After execution, update `scanJobRepo.UpdateNextRunAt(ctx, scheduleID, nextRun)` based on the schedule frequency

Register with gocron using:
- `gocron.WithSingletonMode(gocron.LimitModeReschedule)` — singleton mode skips if already running
- `gocron.WithName(key)` — named for debugging

Store the gocron.Job in activeJobs map.

**UpdateSchedule(ctx, schedule entity.ScanSchedule) error:**
1. Remove existing gocron job if present: look up key in activeJobs, call `s.scheduler.RemoveJob(job.ID())`
2. If schedule.IsEnabled, call `registerSchedule(schedule)` to re-register with new settings
3. If not enabled, just remove (don't re-register)
4. Persist to DB: `scanJobRepo.UpsertSchedule(ctx, &schedule)`

**RemoveSchedule(ctx, orgID, entityType string) error:**
1. Remove gocron job from activeJobs
2. Delete from DB: `scanJobRepo.DeleteSchedule(ctx, orgID, entityType)`

**TriggerManualScan(ctx context.Context, orgID, entityType string) (string, error):**
Per CONTEXT.md: "Manual Run Now button to trigger scans on demand"
1. Check if already running for this entity: `scanJobRepo.GetRunningJobForEntity(ctx, orgID, entityType)`. If running, return error "scan already running for this entity".
2. Call `scanService.ExecuteScan(ctx, orgID, entityType, "manual", nil)` in a goroutine with panic recovery.
3. Return the jobID immediately (async execution).

**GetActiveSchedules() []string:** Return list of active schedule keys for debugging.

**calculateNextRun(schedule entity.ScanSchedule) time.Time:** Helper to compute the next run time from current time + frequency configuration.

IMPORTANT:
- The scheduler runs in-process (not a separate worker). It starts in main.go and shuts down gracefully.
- Use `context.Background()` for scheduled task execution (not tied to any HTTP request).
- gocron's SingletonMode handles the "skip if already running" requirement at the scheduler level. The check in the task function is a safety net for manual triggers and race conditions.
- Log all schedule registrations and skip events for debugging.
