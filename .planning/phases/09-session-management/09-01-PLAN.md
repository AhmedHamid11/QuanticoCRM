1. Create `middleware/session_timeout.go`:
   ```go
   type SessionTimeoutConfig struct {
       AuthService *service.AuthService
       // Endpoints to skip activity tracking (polling, health checks)
       SkipActivityUpdate []string
   }

   func NewSessionTimeoutMiddleware(config SessionTimeoutConfig) fiber.Handler
   ```

   Middleware logic:
   - Get session from JWT claims (already validated by auth middleware)
   - Load session from database to get timeout values
   - Check absolute timeout: `now - session.CreatedAt > AbsoluteTimeoutMinutes`
   - Check idle timeout: `now - session.LastActivityAt > IdleTimeoutMinutes`
   - If either exceeded: return 401 with `{"error": "Session expired", "code": "SESSION_EXPIRED"}`
   - If not expired AND path not in SkipActivityUpdate: update LastActivityAt
   - SkipActivityUpdate should include: `/auth/refresh`, `/auth/extend-session`

2. Add to `repo/session.go` (or auth.go if session queries live there):
   - `GetSessionByUserAndOrg(ctx, userID, orgID) (*Session, error)` - get active session
   - `UpdateLastActivity(ctx, sessionID) error` - set last_activity_at = now

3. Update `service/auth.go`:
   - On login/register: Set idle_timeout_minutes and absolute_timeout_minutes from defaults (30, 1440)
   - Add `GetSessionWithTimeouts(ctx, userID, orgID) (*entity.Session, error)` for middleware
   - Add `ExtendSession(ctx, userID, orgID) error` - updates last_activity_at

Per CONTEXT.md: Only clicks and typing reset timer on frontend. Backend activity updates only happen for user-initiated requests (not polling).
