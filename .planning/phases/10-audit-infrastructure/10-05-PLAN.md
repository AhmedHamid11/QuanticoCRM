```go
package handler

import (
    "encoding/csv"
    "encoding/json"
    "time"

    "github.com/fastcrm/backend/internal/repo"
    "github.com/fastcrm/backend/internal/util"
    "github.com/gofiber/fiber/v2"
)

type AuditHandler struct {
    repo *repo.AuditRepo
}

func NewAuditHandler(repo *repo.AuditRepo) *AuditHandler {
    return &AuditHandler{repo: repo}
}
```

Implement endpoints:

1. List - GET /admin/audit-logs
   - Parse query params: eventTypes (comma-separated), userId, dateFrom, dateTo, page, pageSize
   - Get orgID from context
   - Call repo.List with filters
   - Return paginated response

2. Export - GET /admin/audit-logs/export
   - Same filters as List
   - Query param: format=csv|json (default json)
   - For CSV: Set Content-Type: text/csv, Content-Disposition: attachment
   - For JSON: Return array of entries
   - No pagination (return all matching, with reasonable limit like 10000)

3. VerifyChain - GET /admin/audit-logs/verify
   - Get orgID from context
   - Call repo.VerifyChainIntegrity
   - Return: { valid: bool, entriesVerified: int, errors: [] }

4. GetEventTypes - GET /admin/audit-logs/event-types
   - Return list of all event types for filter dropdown
   - Static list from entity constants

Register routes in main.go:
```go
auditHandler := handler.NewAuditHandler(auditRepo)
adminGroup.Get("/audit-logs", auditHandler.List)
adminGroup.Get("/audit-logs/export", auditHandler.Export)
adminGroup.Get("/audit-logs/verify", auditHandler.VerifyChain)
adminGroup.Get("/audit-logs/event-types", auditHandler.GetEventTypes)
```
