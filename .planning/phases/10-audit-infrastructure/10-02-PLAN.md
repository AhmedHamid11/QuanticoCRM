2. Login handler - add audit after success/failure:
   ```go

   go h.auditLogger.LogLoginAttempt(c.Context(), input.Email, ipAddress, userAgent, true, "")

   // On FAILURE (in handleAuthError or before returning error):
   // Need to call audit with success=false
   ```

   For login failures, add audit call before returning error:
   - In the error path (line 92-94), call:
     `go h.auditLogger.LogLoginAttempt(c.Context(), input.Email, ipAddress, userAgent, false, err.Error())`

3. Logout handler - add audit:
   ```go
   // After successful logout (around line 153-155):
   if userID, ok := c.Locals("userID").(string); ok {
       email, _ := c.Locals("email").(string)
       orgID, _ := c.Locals("orgID").(string)
       go h.auditLogger.LogLogout(c.Context(), userID, email, orgID, c.IP(), c.Get("User-Agent"))
   }
   ```

4. ChangePassword handler - add audit:
   ```go
   // After successful password change (around line 455-457):
   orgID, _ := c.Locals("orgID").(string)
   email, _ := c.Locals("email").(string)
   go h.auditLogger.LogPasswordChange(c.Context(), userID, email, orgID, c.IP())
   ```

5. ResetPassword handler - add audit (optional but good):
   ```go
   // After successful reset (around line 545-547):

       EventType: service.AuditEventPasswordReset,
       IPAddress: c.IP(),
       UserAgent: c.Get("User-Agent"),
       Success:   true,
   })
   ```
