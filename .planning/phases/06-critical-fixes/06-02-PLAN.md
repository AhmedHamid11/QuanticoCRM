<task type="auto">
  <name>Task 1: Create dedicated auth rate limiter middleware</name>
  <files>FastCRM/fastcrm/backend/internal/middleware/ratelimit.go</files>
  <action>
Create a dedicated rate limiting middleware for auth endpoints:

1. **AuthRateLimiterConfig struct:**
   - Max: int (default 5)
   - Window: time.Duration (default 1 minute)

2. **NewAuthRateLimiter(config AuthRateLimiterConfig) fiber.Handler:**
   - Uses in-memory storage (sync.Map with IP -> request count + window start)
   - Key generation: Use X-Forwarded-For if present, else c.IP()
   - On limit reached, return 429 with JSON body:
     ```json
     {
       "error": "Too many authentication attempts",
       "message": "Rate limit exceeded. Please wait before trying again.",
       "retry_after": 60
     }
     ```
   - Include Retry-After header: `Retry-After: 60`

3. **DefaultAuthRateLimiterConfig() AuthRateLimiterConfig:**
   - Max: 5 (from CONTEXT.md)
   - Window: 1 * time.Minute

4. **Cleanup goroutine:**
   - Run background cleanup every 5 minutes to remove expired entries
   - Prevents memory leak from abandoned rate limit entries

Why custom instead of fiber/limiter: Need consistent JSON response format across all auth endpoints, and existing login limiter duplicates what we want globally.

Note: This is in-memory, so rate limits reset on restart and are per-instance. Per CONTEXT.md, this is acceptable.
  </action>
