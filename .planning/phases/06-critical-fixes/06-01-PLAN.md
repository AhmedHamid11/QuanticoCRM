<task type="auto">
  <name>Task 2: Create security middleware for CORS and HSTS</name>
  <files>FastCRM/fastcrm/backend/internal/middleware/security.go</files>
  <action>
Create security middleware that:

1. **HSTS() middleware:**
   - Add header: `Strict-Transport-Security: max-age=31536000; includeSubDomains`
   - Apply to ALL responses
   - 31536000 seconds = 1 year

2. **NewCORSConfig(allowedOrigins []string, isDev bool) cors.Config:**
   - If isDev: Allow localhost:* patterns automatically
   - In production:
     - Parse ALLOWED_ORIGINS into exact match list
     - Support Vercel preview URLs pattern (*.vercel.app) if configured
   - AllowCredentials: true (needed for HttpOnly cookies in Phase 07)
   - AllowMethods: GET,POST,PUT,PATCH,DELETE,OPTIONS
   - AllowHeaders: Origin,Content-Type,Accept,Authorization

3. **Custom CORS handler (not using fiber's cors middleware directly):**
   - Check Origin header against allowlist
   - If origin not in allowlist AND not development mode:
     - Log blocked origin: `log.Printf("[CORS] Blocked origin: %s", origin)`
     - Return response WITHOUT CORS headers (silent reject per CONTEXT.md)
   - If origin allowed: Set Access-Control-Allow-Origin to specific origin (not *)

Why custom: Fiber's cors middleware always returns CORS headers. We need silent reject behavior.
  </action>

Run: `cd FastCRM/fastcrm/backend && go build ./internal/middleware`
Confirm: Build succeeds without errors
  </verify>
