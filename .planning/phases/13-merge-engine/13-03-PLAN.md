**ExecuteMerge(ctx, db *sql.DB, orgID, userID string, req entity.MergeRequest) (*entity.MergeResult, error)**

This is the atomic merge operation. All steps happen within a single SQLite transaction.

Per user decisions:
- All related records transfer automatically (no cherry-picking)
- If both survivor and duplicate have same related record, keep both
- Duplicate is archived with survivor reference
- Snapshot stored for 30-day undo

Steps (all within transaction):

1. **Validate inputs**: Ensure survivorID and duplicateIDs are provided, entity type exists
2. **Ensure archive columns**: Call `repo.EnsureArchiveColumns(ctx, db, tableName)` to add archived_at/archived_reason/survivor_id columns if missing
3. **Ensure merge_snapshots table**: Call `mergeRepo.EnsureTableExists(ctx)` defensively
4. **Begin transaction**: `db.BeginTx(ctx, nil)` with deferred rollback on panic (matching bulk.go pattern)
5. **Snapshot survivor pre-merge state**: Use `util.FetchRecordAsMap(ctx, db, tableName, survivorID, orgID)` — but since we're in a transaction, use `tx` not `db`. Write a helper `fetchRecordAsMapTx(ctx, tx, tableName, id, orgID)` that uses tx.QueryContext instead. Store as JSON.
6. **Snapshot each duplicate**: Same approach, store as JSON array
7. **Discover related records for each duplicate**: Call `discoveryService.DiscoverRelatedRecords(ctx, db, orgID, entityType, dupID)` — note: discovery queries run OUTSIDE the transaction (read-only metadata queries). The actual FK updates run inside the transaction.
8. **Transfer FK references** (inside tx): For each related record group, for each record:
   - Build UPDATE statement: `UPDATE {table} SET {fk_column} = ? WHERE id = ? AND org_id = ?`
   - Set FK to survivorID
   - Track the change in FKChange struct for snapshot (recordId, fkField, oldValue=duplicateID)
   - Use `util.GetTableName(group.EntityType)` for table name
   - Use `util.CamelToSnake(group.FKField)` for column name (the FKField is stored as camelCase like "accountId" -> needs to become "account_id")
9. **Update survivor fields** (inside tx): Build SET clauses from req.MergedFields, similar to bulk.go updateRecord pattern.
   - Get field definitions for the entity type via metadataRepo
   - For each field in mergedFields: build SET clause using the same field handling as bulk.go (handle link fields with _id/_name pattern, regular fields with CamelToSnake)
   - Add `modified_at = ?` and `modified_by_id = ?`
   - Execute UPDATE
10. **Archive each duplicate** (inside tx, AFTER FK transfer per Pitfall 2):
    - `UPDATE {table} SET archived_at = ?, archived_reason = 'MERGED', survivor_id = ? WHERE id = ? AND org_id = ?`
    - archived_at = time.Now().UTC().Format(time.RFC3339)
11. **Save snapshot** (inside tx): Build MergeSnapshot with all JSON fields, insert via direct SQL in transaction (not through repo which uses a separate connection)
12. **Commit transaction**
13. **Create audit log** (OUTSIDE transaction, fire-and-forget via auditLogger): Log RECORD_MERGE event with survivor_id, duplicate_ids, snapshot_id, entity_type
14. **Cleanup expired snapshots** (OUTSIDE transaction, opportunistic): Call `mergeRepo.CleanupExpired(ctx, orgID)` — non-critical, ignore errors
15. Return MergeResult with survivorID, snapshotID, mergedAt

Transaction error handling: Any error in steps 5-11 triggers tx.Rollback() and returns the error. This guarantees atomicity per MERGE-06.

**UndoMerge(ctx, db *sql.DB, orgID, userID, snapshotID string) error**

Atomic undo operation. All steps within a single transaction.

Per user decision: "User can undo merge within 30 days, restoring source records and re-pointing related records."

Steps:

1. **Fetch snapshot**: Get by ID and orgID. Validate:
   - Snapshot exists
   - Not expired (ExpiresAt > now)
   - Not already consumed (ConsumedAt is nil)
2. **Begin transaction** with deferred rollback
3. **Restore survivor to pre-merge state**: Parse SurvivorBefore JSON, build UPDATE statement with all field values, execute
4. **Un-archive each duplicate**: Parse DuplicateSnapshots JSON, for each:
   - `UPDATE {table} SET archived_at = NULL, archived_reason = NULL, survivor_id = NULL WHERE id = ? AND org_id = ?`
   - Restore duplicate fields from snapshot
5. **Revert FK references**: Parse RelatedRecordFKs JSON, for each entity type, for each FK change:
   - `UPDATE {table} SET {fk_column} = ? WHERE id = ? AND org_id = ?` (set back to oldValue = original duplicate ID)
6. **Mark snapshot consumed**: `UPDATE merge_snapshots SET consumed_at = ? WHERE id = ? AND org_id = ?`
7. **Commit transaction**
8. **Audit log** (outside tx): Log MERGE_UNDO event

func (a *AuditLogger) LogRecordMerge(ctx context.Context, actorID, orgID, entityType, survivorID, snapshotID string, duplicateIDs []string) {
    a.Log(ctx, AuditEvent{
        EventType: AuditEventRecordMerge,
        ActorID:   actorID,
        TargetID:  survivorID,
        OrgID:     orgID,
        Success:   true,
        Details: map[string]interface{}{
            "entityType":   entityType,
            "survivorId":   survivorID,
            "duplicateIds": duplicateIDs,
            "snapshotId":   snapshotID,
        },
    })

        TargetID:  survivorID,
        OrgID:     orgID,
