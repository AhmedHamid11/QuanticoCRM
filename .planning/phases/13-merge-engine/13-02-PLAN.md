Create `backend/internal/repo/merge.go` following the existing repo patterns (see audit.go, metadata.go).

The MergeRepo should:
1. Use `db.DBConn` interface (not raw *sql.DB) for multi-tenant support
2. Implement `WithDB(conn db.DBConn) *MergeRepo` for tenant database routing
3. Implement `EnsureTableExists(ctx)` defensively (like AuditRepo pattern)

Methods:

**Save(ctx, snapshot *entity.MergeSnapshot) error**
- INSERT merge snapshot into merge_snapshots table
- All JSON fields (survivor_before, duplicate_ids, duplicate_snapshots, related_record_fks) are stored as TEXT

**GetByID(ctx, orgID, snapshotID string) (*entity.MergeSnapshot, error)**
- Fetch by id and org_id (multi-tenant isolation)
- Return nil, nil if not found (sql.ErrNoRows pattern)

**GetBySurvivor(ctx, orgID, survivorID string) ([]entity.MergeSnapshot, error)**
- Fetch all snapshots for a survivor record (for merge history)
- Order by created_at DESC
- Only return non-consumed snapshots (consumed_at IS NULL)

**ListByOrg(ctx, orgID string, page, pageSize int) ([]entity.MergeSnapshot, int, error)**
- Paginated list of all merge snapshots for an org
- Returns total count for pagination
- Order by created_at DESC

**MarkConsumed(ctx, orgID, snapshotID string) error**
- UPDATE merge_snapshots SET consumed_at = NOW() WHERE id = ? AND org_id = ?
- Used when undo is performed to prevent double-undo

**CleanupExpired(ctx, orgID string) (int, error)**
- DELETE FROM merge_snapshots WHERE org_id = ? AND expires_at < datetime('now') AND consumed_at IS NULL
- Returns count of deleted snapshots
- Called opportunistically before new merges

**EnsureArchiveColumns(ctx, db db.DBConn, tableName string) error**
- Static utility method (doesn't need receiver)
- Checks PRAGMA table_info for archived_at, archived_reason, survivor_id columns
- Adds missing columns via ALTER TABLE
- Uses the same defensive pattern as util.SyncFieldColumns (ignore "duplicate column" errors)
- This is called by the merge service before archiving a record

Implementation notes:
- Use `time.RFC3339` format for all timestamp fields (consistent with existing codebase)
- Follow the `db.DBConn` interface pattern from audit.go
- Scan time fields as strings then parse (existing pattern in metadata.go)

  <action>
Create `backend/internal/service/merge_discovery.go` that discovers related records dynamically.

Per user decision: "All related records (Tasks, Notes, Activities, lookup references) transfer to survivor automatically. No cherry-picking."

The discovery service needs access to:
- `MetadataRepo` for ListEntities and ListFields
- A database connection for querying actual related records

**MergeDiscoveryService struct:**
```go
type MergeDiscoveryService struct {
    metadataRepo *repo.MetadataRepo
}

func NewMergeDiscoveryService(metadataRepo *repo.MetadataRepo) *MergeDiscoveryService
```

**DiscoverRelatedRecords(ctx, db db.DBConn, orgID, entityType, recordID string) ([]entity.RelatedRecordGroup, error)**

Algorithm:
1. Get all entities for the org via `metadataRepo.ListEntities(ctx, orgID)`
2. For each entity, get fields via `metadataRepo.ListFields(ctx, orgID, entity.Name)`
3. Find fields where `field.Type == entity.FieldTypeLink` and `field.LinkEntity != nil && *field.LinkEntity == entityType`
4. For each matching lookup field:
   - Build table name: `util.GetTableName(entity.Name)`
   - Build FK column name: `util.CamelToSnake(field.Name) + "_id"` (lookup fields store FK as `{field_name}_id`)
   - Query: `SELECT id FROM {table} WHERE org_id = ? AND {fk_column} = ? AND (archived_at IS NULL OR archived_at = '')`
   - Note: Include the `archived_at IS NULL OR archived_at = ''` clause defensively — the column may not exist yet (will be added at merge time), so wrap in a fallback: try with archived_at filter first, if column doesn't exist, query without it
   - Build RelatedRecordGroup with entity.Name, field.Name + "Id" as FKField (camelCase for consistency), and found record IDs
5. Return all groups (even empty ones are fine to exclude)
6. Skip the entity being merged (don't discover self-references as "related records")

**CountRelatedRecords(ctx, db db.DBConn, orgID, entityType string, recordIDs []string) ([]entity.RelatedRecordCount, error)**

For the merge preview, count related records per entity type for EACH record being merged:
1. Call DiscoverRelatedRecords for each recordID
2. Aggregate into RelatedRecordCount structs with entity type, label, count, and record ID
3. Also fetch the actual related records (up to 20 per entity type per record) for the expandable preview list. Query `SELECT id, {display_field} as name FROM {table} WHERE org_id = ? AND {fk_col} = ?` with the entity's display_field from EntityDef.

**CalculateCompleteness(record map[string]interface{}) float64**

Per user decision: "Completeness score should be visible so the suggestion is transparent."
- Count non-empty, non-null fields (excluding system fields: id, orgId, createdAt, modifiedAt, createdById, modifiedById, archivedAt, archivedReason, survivorId)
- Divide by total non-system fields
- Return 0.0-1.0

**SuggestSurvivor(records []map[string]interface{}) string**

Per user decision: "System auto-suggests the most complete record as survivor."
- Calculate completeness for each record
- Return the ID of the record with highest completeness score
- Tie-break: prefer the record with earlier createdAt (older record)

Implementation notes:
- Pass `db.DBConn` to discovery methods (not stored on struct) because the tenant DB is resolved per-request
- The `metadataRepo` can use `.WithDB()` to switch to tenant DB when needed — but since discovery is called from the merge service which already has tenant context, pass the tenant-aware metadataRepo
- Use `util.GetTableName()` for table name conversion
- Use `util.CamelToSnake()` for column name conversion
